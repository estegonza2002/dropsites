# DropSites v1.0 Changelog

Released: March 30, 2026

## New Features

### Phase 1 — Core Platform
- Static site publishing: drop a file or folder, get a shareable URL
- Support for HTML, ZIP, JS bundles, PDFs, and any browser-renderable content
- Magic link, Google, and GitHub OAuth authentication
- Workspace management with owner/publisher/viewer roles
- Password protection with brute-force rate limiting
- Link expiry with reactivation support
- CodeMirror in-browser code editor with conflict detection
- Auto-navigation widget injection at serve time
- Share sheet: copy link, QR code, embed snippet, email share
- Transactional email (Resend) and SMS (Twilio) notifications
- Analytics: views, unique visitors, referrers, time-series charts
- Bandwidth tracking with quota alerts and CSV export
- Audit log for all key operations
- Compression, CDN cache headers, and Lighthouse performance ≥ 90
- WCAG 2.1 AA accessibility compliance
- 14-day Pro trial with countdown
- Bot filtering, abuse reports, DMCA/TOS pages, content hash registry

### Phase 2 — API & Integrations
- REST API v1 with OpenAPI spec
- API key management with lifecycle controls
- Per-minute/daily/monthly rate limits with burst support
- Single-file PATCH endpoint with version history
- Custom domain support with ACME TLS provisioning
- Per-recipient access tokens with view limits
- Webhooks with HMAC signing, retry logic, and delivery log
- Version history: last 3 versions, preview, restore
- Team namespace URL prefixes
- Stripe billing: monthly/annual plans, dunning, grace periods
- Offline licence keys with air-gapped mode
- Helm chart and Terraform modules for GCP/AWS/Azure
- S3/GCS/Azure/MinIO/local storage backends
- Safe Browsing + VirusTotal scanning with quarantine queue
- Geographic and device analytics with comparison mode
- CORS control per deployment
- Workspace transfer, activity feed, default settings
- JavaScript SDK (`@dropsites/sdk`)
- CLI tool (`@dropsites/cli`)
- GitHub Actions action
- Daily backup with disaster recovery runbook
- SLA document with 90-day uptime history
- Preview thumbnails via headless browser
- Analytics PDF export
- Global cross-workspace search (Cmd+K)

### Phase 3 — AI Integration
- MCP server (`@dropsites/mcp`) for Claude Desktop and claude.ai
- `deploy_site`, `list_sites`, `delete_site`, `update_site` tools
- Setup guide and `/mcp` marketing page
- Closed beta invite system with structured feedback collection

## Bug Fixes

- Fixed path traversal vulnerability in file serving middleware
- Fixed slug collision edge case in auto-generation
- Fixed rate limit bypass via header manipulation

## Improvements

- Reduced cold-start serving latency by 40% with R2 edge caching
- Improved ZIP extraction to handle nested archives correctly
- Dashboard search now debounces at 200ms for smoother UX
